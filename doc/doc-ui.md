# Документация UI React Blog (Интерфейс, Создающий Ценность)

### 1. Общая Информация: UI как Лицо Системы

**Название проекта:** React Blog System

**Описание:** Этот документ описывает пользовательский интерфейс (UI) приложения. UI — это **лицо нашей системы**, через которое пользователи взаимодействуют с контентом и управляют им. Мы стремимся создать интерфейс, который будет **интуитивно понятным, эффективным и отзывчивым**, следуя принципам **чистоты кода** и **разделения ответственности**.

**Цель:** Предоставить четкое описание структуры экранов, принципов навигации и используемых компонентов. Это обеспечивает **единообразие** и **качество** пользовательского опыта, что критически важно для профессионального продукта.

**Принципы, отраженные в UI:**
*   **Создание Ценности:** Фокус на контенте (постах) и удобстве его потребления.
*   **Качество > Количество:** Минималистичный дизайн, где каждый элемент имеет функцию.
*   **Система и Оптимизация:** Использование переиспользуемых компонентов (атомарный дизайн) и оптимизированного рендеринга (SWR).
*   **Отзывчивость:** Мгновенная реакция на действия пользователя (оптимистичные обновления, скелетоны загрузки).

### 2. Основные Экраны: Путь Пользователя по Системе

Основные экраны представляют собой ключевые точки взаимодействия пользователя с системой React Blog.

1.  **Главная Страница (Home Page / Feed):**
    *   **Цель:** Предоставить пользователю обзор всех доступных публикаций и возможность быстрого поиска.
    *   **Элементы:**
        *   **Header:** Логотип, Строка поиска (Search Bar), Кнопка "Create Post".
        *   **Список Постов:** Сетка или список карточек постов.
        *   **Состояния:** Загрузка (Skeletons), Ошибка (Error Toast), Пустой список (Empty State).
    *   **Взаимодействие:** Клик по карточке ведет на детальную страницу. Ввод в поиск фильтрует список в реальном времени (через Zustand).

2.  **Страница Просмотра Поста (Post Details):**
    *   **Цель:** Комфортное чтение полного контента и управление записью.
    *   **Элементы:**
        *   **Навигация:** Кнопка "Назад" (<).
        *   **Контент:** Заголовок (H1), Мета-данные (Автор, Дата), Тело поста (Markdown или текст).
        *   **Управление:** Панель действий (Редактировать, Удалить).
        *   **Комментарии:** Секция обсуждения (список + форма добавления).
    *   **Взаимодействие:** Удаление вызывает модальное окно подтверждения. Редактирование переводит на форму.

3.  **Страница Создания/Редактирования (Create/Edit Post):**
    *   **Цель:** Удобный интерфейс для генерации контента.
    *   **Элементы:**
        *   **Заголовок:** "Create New Post" или "Edit Post".
        *   **Форма:** Поля Title (Input), Author (Input), Content (Textarea).
        *   **Валидация:** Визуальные подсказки об ошибках (красная рамка, текст под полем) через React Hook Form.
        *   **Действия:** Кнопки "Publish/Update" (Primary) и "Cancel" (Secondary).

### 3. Структура Интерфейса: Построение на Системе Компонентов

UI построен на **модульной системе компонентов**, что обеспечивает **единообразие** и **скорость разработки**. Мы используем Tailwind CSS для быстрой и системной стилизации.

*   **Атомы (Базовые элементы):**
    *   **Button:** Основной интерактивный элемент. Варианты: `Primary` (синий), `Secondary` (серый), `Danger` (красный для удаления).
    *   **Input / Textarea:** Поля ввода с поддержкой состояний (focus, error, disabled).
    *   **Badge:** Метки для категорий или статусов.
    *   **Spinner / Skeleton:** Индикаторы загрузки для улучшения UX (вместо белого экрана).

*   **Молекулы (Составные элементы):**
    *   **PostCard:** Компонент превью поста. Содержит заголовок, отрывок текста, дату и автора.
    *   **SearchBar:** Поле ввода с иконкой лупы и кнопкой очистки.
    *   **Modal:** Окно подтверждения действий (например, при удалении).

*   **Организмы (Сложные блоки):**
    *   **PostList:** Управляет рендерингом списка карточек, обрабатывает состояния загрузки и пустого результата.
    *   **PostForm:** Форма с логикой валидации (Zod) и обработки отправки.
    *   **Navbar:** Глобальная навигация и брендинг.

### 4. Принципы Навигации: Понятный Путь

Навигация спроектирована так, чтобы быть **интуитивно понятной** и следовать стандартам SPA.

*   **Глобальная навигация:** Navbar доступен на всех страницах. Логотип ведет на Главную.
*   **Хлебные крошки (визуально):** На странице поста кнопка "Назад" возвращает к списку.
*   **Редиректы:**
    *   После создания поста -> Переход на Главную (или на созданный пост).
    *   После удаления поста -> Переход на Главную.
    *   При попытке открыть несуществующий пост -> Страница 404.

### 5. Обработка Состояний и Ошибок (UX)

Мы уделяем особое внимание тому, как интерфейс реагирует на **нештатные ситуации** и **процессы ожидания**, так как это напрямую влияет на восприятие **качества**.

*   **Загрузка (Loading):**
    *   **Первичная загрузка:** Показываем Skeleton-лоадеры, имитирующие структуру карточек. Это снижает когнитивную нагрузку (пользователь уже видит структуру).
    *   **Отправка формы:** Кнопка "Submit" переходит в состояние `disabled` и показывает спиннер. Блокируется повторная отправка.

*   **Ошибки (Errors):**
    *   **Сетевые ошибки:** Всплывающее уведомление (Toast) с понятным текстом ("Failed to load posts. Please try again.").
    *   **Ошибки валидации:** Текст ошибки появляется прямо под полем ввода (Inline Validation) в реальном времени (mode: `onBlur` или `onChange`).

*   **Пустые состояния (Empty States):**
    *   Если постов нет (или поиск не дал результатов), показываем дружелюбную иконку и текст "No posts found", возможно, с кнопкой "Create first post".

### 6. Адаптивность (Responsive Design)

Интерфейс полностью адаптивен и следует принципу **Mobile-First**.

*   **Mobile (< 640px):**
    *   Сетка постов: 1 колонка.
    *   Кнопки: На всю ширину экрана для удобства нажатия пальцем.
    *   Навигация: Упрощенная.
*   **Tablet (640px - 1024px):**
    *   Сетка постов: 2 колонки.
*   **Desktop (> 1024px):**
    *   Сетка постов: 3 или 4 колонки.
    *   Контент центрирован с максимальной шириной (`max-w-7xl`).

### 7. Примечания: Технологический Контекст

*   **SWR Integration:** UI тесно связан с SWR. Индикаторы загрузки напрямую зависят от флага `isLoading` из хука `useSWR`. Оптимистичные обновления (Optimistic UI) могут использоваться для мгновенного отклика при лайках или удалении (визуальное удаление до ответа сервера).
*   **Zustand Store:** Состояние поиска (строка в SearchBar) хранится в глобальном сторе Zustand, что позволяет фильтровать список постов без лишних пропсов (prop drilling).
