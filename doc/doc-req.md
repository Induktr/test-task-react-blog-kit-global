# Документация Требований (Requirements)

### 1. Функциональные Требования (FR)

#### 1.1. Просмотр Контента
*   **FR-01:** Пользователь должен видеть список всех постов на главной странице.
*   **FR-02:** Карточка поста должна содержать заголовок, краткое описание и дату.
*   **FR-03:** Пользователь может перейти на детальную страницу поста по клику.

#### 1.2. Управление Контентом (CRUD)
*   **FR-04:** Пользователь может создать новый пост (Title, Body, Author).
*   **FR-05:** Пользователь может редактировать существующий пост.
*   **FR-06:** Пользователь может удалить пост (с подтверждением).

#### 1.3. Поиск и Фильтрация
*   **FR-07:** Пользователь может искать посты по заголовку (фильтрация в реальном времени).
*   **FR-08:** Строка поиска должна сохранять свое состояние при навигации (через Zustand).

### 2. Нефункциональные Требования (NFR)

*   **NFR-01 (Performance):** Использование стратегии `Stale-While-Revalidate` (SWR) для мгновенного отображения кэшированных данных.
*   **NFR-02 (UX):** Индикация загрузки (Skeletons/Spinners) и обработка ошибок (Toasts/Error Boundaries).
*   **NFR-03 (Quality):** Отсутствие `any` в TypeScript. Валидация форм через схему Zod.

### 3. Ограничения (Constraints)
*   Дедлайн реализации: 48 часов.
*   Стек строго ограничен требованиями тестового задания (SWR, Firebase).
